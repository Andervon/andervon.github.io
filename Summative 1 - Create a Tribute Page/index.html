Computer Servers and DNS (Domain Name System)

Computer Servers:

A computer server or known more simply as a server, is a type of computer just like your desktop, laptop, or phone. Like the name suggests, servers serve other computers and help them do tasks they could not do. There are many types of servers that do different things and date back to the earliest days of computing. Servers were created for a variety of reasons, but one of the main reasons was to store data or information that could be acceded by other computers. This model is still largely how servers work to this very day! While any computer (such as a phone) could act as a server, these machines are not designed to operate such high workloads. A dedicated server typically runs a server program which works to help it complete its tasks or requests by computers. Two examples are application servers which run web apps (a web app is an application that does not run on your computer. Instead it communicates with an application sever to do all its tasks) like twitter and web servers which host websites ("hosting" is when you store all the information a website needs to work). There are also mail servers which handle email and send the messages to and from different computers. The most common types of server are database servers which simply hold data. These servers are what makes up cloud services such as iCloud, Google Drive, or Dropbox.

In summary, a server is just a special type of computer designed to do special tasks. They then communicate and work with other less powerful computers to get more things done than these devices could expect to do themselves.

DNS (Domain Name System):

The DNS is a system to store and locate information about websites more easily. Instead of having to remember the IP address of a web server (IP is just a special name for every computer) in which a website is located you can just remember its url. An example of this is that you can remember www.google.com instead of 172.217.2.196 (which is the IP of the google.com server). You can think of DNS like the street address to your house and an IP address as its coordinates on the globe. While they are both the exact same thing, the address is much easier to remember than the coordinates of your house. Before DNS, the people that ran the early internet had a document called HOSTS.TXT in which they wrote down the IP of every computer! Due to the number of sites growing, this had to change so Paul Mockapetris was tasked to develop a new system for storing IP addresses. In 1983, DNS was created and became one of the original Internet Standards.


What Mockapetris created was the idea of domains (the little things at the end of URLs like .com, .net, .org, .uk or .edu). This idea allowed for websites to be categorized and stored in a way that they could quickly accessed. Instead of remembering the whole IP, you only needed the URL with a domain at the end.

How it works is that when you want to go to http://example.com/, your computer (1) first checks to see if it already knows the location of this site (IP address information is stored on your computer for a limited time so if you need to visit the same site multiple times a day it is faster). If it does not have it stored, your computer asks the Resolving Name Server (2) which is often operated by your ISP and stores information about domains and IP addresses. The first thing this server does is it checks to see if it has information already stored. If it does, it sends the result to your computer which connects to the website. If it doesn't then it moves along to the Root Name Server (3). The root name server then refers the Resolving Name Server to a Top Level Domain server (4) (The specific one depends on the domain of the site. in this example it does to the .com TLD server). This domain name server then will refer you to an authoritative name server which actually knows the location or IP address of the site you wish to visit. The resolving name server takes that information and stores the IP address of this website before sending it to your computer. Once your computer has the IP address, you can connect to the server that hosts the site of http://example.com/.

What is amazing is that this all happens in less than a second! Thanks to Paul Mockapetris' idea to create a DNS, you can easily connect to the millions of websites without having to remember the long strings of characters that actually represent the servers they are stored on.
